## CONTEXTO SGIND — REGLA FUNDAMENTAL

```
REGLA CRÍTICA DE GOBERNANZA:
╔════════════════════════════════════════════════════════════════════════╗
║ CADA INDICADOR DEBE TENER UNA ÚNICA FÓRMULA DEFINIDA Y TRAZABLE      ║
║ - Una única definición en docs/core/02_Logica_Indicadores.md         ║
║ - Una única implementación en core/calculos.py                        ║
║ - Una única línea base documentada                                    ║
║ - Una única meta o criterio de éxito                                  ║
║ - Un único responsable asignado                                       ║
║ - Un único período de actualización declarado                         ║
║ - Una única fuente de datos trazable                                  ║
╚════════════════════════════════════════════════════════════════════════╝
```

**Impacto de Violaciones:**
- Múltiples fórmulas → Inconsistencia en reportes
- Fórmulas sin documentación → Imposible auditar cambios
- Fórmulas diferentes en frontend vs backend → Desconfianza en datos
- Cambios sin registro → Imposible reproducir cálculos históricos
- Indicadores sin línea base → Imposible evaluar progreso

### 2. AUDITORÍA DE FÓRMULAS

**Verificar para cada indicador:**

**A. Unicidad de Fórmula**
- ¿Existe exactamente UNA definición en docs/core?
- ¿Existe exactamente UNA implementación en código?
- ¿Coinciden matemáticamente docs vs código?
- Ejemplo válido:
  ```
  docs: "Cumplimiento = (Cumplidos / Total) * 100"
  code: df['Cumplimiento'] = (df['Cumplidos'] / df['Total']) * 100
  ✅ COINCIDEN
  ```
- Ejemplo INVÁLIDO:
  ```
  docs: "(Cumplidos / Total) * 100"
  code: "(Cumplidos / (Total - NoAplica)) * 100"
  ❌ FÓRMULAS DIFERENTES
  ```

**B. Documentación de Fórmula**
- ¿Está documentada en formato estándar?
- ¿Incluye descripción de variables?
- ¿Incluye rangos esperados?
- ¿Incluye ejemplos de cálculo?
- ¿Registra fecha de última actualización?

**C. Hardcoding**
- ¿Tiene umbrales hardcodeados? (RIESGO: No modificable)
- Ejemplo: `if valor > 0.85:` (debería estar en config)
- Buscar: números literales, constantes sin nombre, magic numbers
- Recomendación: Mover a config/settings.toml

**D. Consistencia Temporal**
- ¿Ha cambiado la fórmula alguna vez?
- ¿Se registraron los cambios?
- ¿Hay datos históricos calculados con fórmula anterior?
- Riesgo: Inconsistencia entre períodos (Enero usa V1, Marzo usa V2)

### 3. AUDITORÍA DE METADATOS

**Línea Base (CRÍTICA)**
- ¿Está definida la línea base?
- ¿En qué período se estableció?
- ¿Está documentada en proyecto?
- Impacto: Sin línea base, imposible medir progreso

**Meta o Criterio de Éxito**
- ¿Está definida la meta?
- ¿Es realista vs histórico?
- ¿Es única o múltiple?
- Impacto: Meta inconsistente = evaluación inconsistente

**Responsable del Indicador**
- ¿Está asignado responsable?
- ¿Tiene contacto documentado?
- Impacto: Sin responsable, nadie es accountable por cambios

**Período de Actualización**
- ¿Es clara la periodicidad?
- ¿Se actualiza realmente en ese período?
- ¿Hay retrasos documentados?
- Impacto: Periodicidad inconsistente = confianza baja

**Fuente de Datos**
- ¿Cuál es la fuente de datos primaria?
- ¿Se permite múltiples fuentes?
- ¿Cómo se reconcilian?
- Impacto: Múltiples fuentes = potencial inconsistencia

### 4. DETECCIÓN DE PROBLEMAS

**PROBLEMA 1: Indicadores Duplicados**
- Mismo indicador calculado múltiples veces
- Ejemplo: "CUMPLIMIENTO" en calculos.py Y "Cumplimiento_%" en generar_reporte.py
- Severidad: CRÍTICA (inconsistencia garantizada)
- Recomendación: Consolidar en módulo único

**PROBLEMA 2: Indicadores Huérfanos**
- Indicador en docs pero no en código (código muerto)
- Indicador en código pero no en docs (falta documentación)
- Severidad: ALTA (imposible auditar)

**PROBLEMA 3: Cambios sin Registro**
- Fórmula cambió pero no está documentado
- Evidencia: Diferencia entre docs vs código
- Severidad: ALTA (imposible reproducción)
- Recomendación: Investigar cambios en git

**PROBLEMA 4: Metas Inconsistentes**
- Meta diferente en doc vs en código
- Meta cambia entre períodos sin justificación
- Severidad: MEDIA (afecta evaluación)

**PROBLEMA 5: Fórmulas sin Línea Base**
- Indicador sin valor inicial documentado
- Imposible medir progreso
- Severidad: MEDIA

**PROBLEMA 6: Indicadores sin Responsable**
- No hay persona accountable
- No hay contacto para preguntas
- Severidad: BAJA (pero IMPORTANTE)

**PROBLEMA 7: Periodicidad Inconsistente**
- Indicador dice "mensual" pero se calcula trimestral
- Severidad: MEDIA (confusión de usuarios)

**PROBLEMA 8: Fuentes Conflictivas**
- Indicador depende de API Kawak AND Excel AND LMI
- Posibilidad de desincronización
- Severidad: MEDIA-ALTA
